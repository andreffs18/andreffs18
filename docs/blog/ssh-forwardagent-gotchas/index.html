SSH ForwardAgent Gotchas

Published February 5, 2019 | #ssh #tips #gotchas | ± 2 mins

So, you’re trying to create an SSH Tunnel to your remote machine to make some gitlab/github clones.

That’s all fine and well until you don’t want to post your password on to the terminal.

First, look at you ~/.ssh/config file and add the “ForwardAgent yes” to that machines ip.

1Host 35.242.133.24
2  IdentityFile ~/.ssh/id_rsa
3  ForwardAgent yes

That says to use your local private key as identification when accessing gitlab on your remote’s machine. Then, just try to check your GitHub/Gitlab connectivity, but first, don’t forget to add you private key to your ssh-agent:

1$ ssh-add -K
2Identity added: /Users/me/.ssh/id_rsa (/Users/me/.ssh/id_rsa)
3$ ssh-add -l
42048 SHA256:vt6nnixTeh5lJpupm8H5+cAuIcxy46LAYdYhfo+WL0E /Users/me/.ssh/id_rsa (RSA)

Now, inside your machine try to check if you can authenticate on Gitlab/Github

1$ ssh -v ubuntu@35.242.133.24
2ubuntu@instance-1:~$ ssh-add -l
32048 SHA256:vt6nnixTeh5lJpupm8H5+cAuIcxy46LAYdYhfo+WL0E /Users/me/.ssh/id_rsa (RSA)
4
5ubuntu@instance-1:~/load-test$ ssh -T git@github.com
6Hi andreffs18! You've successfully authenticated, but GitHub does not provide shell access.
7ubuntu@instance-1:~/load-test$ ssh -T git@gitlab.com
8Welcome to GitLab, @andreffs18!

Now, let’s try to clone our project

1ubuntu@instance-1:~/load-test$ git pull origin master
2Username for 'https://gitlab.com': ^C

What?! I don’t want to put my credentials! Isn’t the ForwardAgent for this?

Okay, let’s look at our remotes:

1ubuntu@instance-1:~/load-test$ git remote -v
2origin    https://gitlab.com/andreffs18/load-test.git (fetch)
3origin    https://gitlab.com/andreffs18/load-test.git (push)

⚠️ If you work with HTTPs urls, it will always ask for your username/password.

If you’re correctly using SSH when cloning / setting remotes. Then make sure you have a ssh-agent to remember your password. That way, you’ll only enter your passphrase once by terminal session.

If it is still too annoying, then simply set a ssh-key without passphrase.

Just change it:

1$ git remote set-url origin git@gitlab.com:andreffs18/load-test.git

And now, on your remote machine you should be able to pull everything from your repo:
